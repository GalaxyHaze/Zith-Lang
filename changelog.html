Release v0.2.0

13 June 2026

Summary

We're excited to share the v0.2.0 release of the Zith compiler! This drop brings major progress across the entire pipeline — from lexer all the way through to MIR. We've introduced a "did you mean?" diagnostic engine that catches typos with Levenshtein-powered suggestions, added dedicated AST nodes for enums, unions, components, traits and interfaces, and layered in a fresh set of parser, sema, type-system, and unification tests. The tooling side also got some love: opencode agents for CI, code review, test writing, and a new changelog command to automate release notes. This is our biggest update yet, and we hope you enjoy the ride!

Highlights

  • Heuristic diagnostic engine — "did you mean?" suggestions for undefined identifiers and type mismatches
  • Dedicated AST node types — EnumDeclNode, UnionDeclNode, ComponentDeclNode, TraitDeclNode, InterfaceDeclNode
  • Expanded test coverage — full parser scan/expression tests, sema-to-HIR lowering, type deduplication, unify errors
  • Comprehensive API guide — new docs/api-guide.md documenting all 8 pipeline stages
  • Opencode automation — agents for CI workflows, code review, compiler review, and test writing
  • Type system internals — type interning, unification, and kind system groundwork
  • Improved diagnostics — better error codes, color output, and emit formatting

Technical Details

Added

  • Heuristic engine with Levenshtein-distance "did you mean?" suggestions (src/sema/heuristic-engine.*)
  • Dedicated AST nodes for enum, union, component, trait, and interface declarations
  • Full parser scan tests and expression-body tests in tests/unit/parser-*.cpp
  • Sema-to-HIR lowering tests (tests/unit/sema-hir.cpp)
  • Type deduplication and unification error tests
  • Opencode agents and skills for CI, code review, test writing, and changelog generation
  • Project documentation: AGENTS.md, ARCH-COMPARISON.md, ARCH-PLAN.md, NEXT_STEPS.md
  • Compiler API guide at docs/api-guide.md
  • Working features example: examples/working-features.zith

Changed

  • AST builder and printer — enum, union, component, trait, interface node support
  • CLI pipeline plan and compilation session — refined stage gating and option handling
  • Diagnostic engine — improved emit logic, error code registry, color support
  • Import manager and resolver — symbol table scope management and module resolution
  • Lexer — keyword table, token kinds, and scanner edge cases
  • Memory layer — arena, dyn-array, optional, result, source-map, string-interning refinements
  • Parser — scan and body expansion passes, recovery heuristics
  • Sema context and pipeline — typing context and function return tracking
  • Type system — type interning, unification, kind IDs, canonical type creation
  • HIR and MIR — instruction types, module structure, lowering passes, verification
  • stdlib — unstable/json.zith updates
  • Build system — CMakeLists, GitHub workflows, install scripts
  • Test common infrastructure — test framework macros and helpers
