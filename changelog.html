Release v0.3.0

13 June 2026

Summary

This release introduces the Name Resolution System — a dedicated pipeline stage between import resolution and semantic analysis. The new Resolver module walks the AST after all imports are known, resolves every identifier to its SymId, and records results in a side table that the Sema pipeline consumes directly. We've also extended the parser with range syntax (..), arrow member access (->), and fat arrows (=>), added a RangeNode to the AST, fixed several diagnostic edge cases, and made the compiler portable to Windows.

Highlights

  • Name Resolution Pipeline Stage — new Resolve stage between import and sema, with qualified and unqualified name resolution
  • Alias Resolution — import aliases (import foo as f) are resolved through the symbol table with target tracking
  • Improved Diagnostics — specific error codes for non-namespace access (NotNamespace) and missing members (NoMember)
  • Parser Extensions — range operator .., arrow access ->, fat arrow =>
  • AST RangeNode — new expression node with lhs/rhs sub-expressions, wired through builder and printer
  • Windows Support — _WIN32 guards added to 5 CLI command files for cross-platform compatibility
  • Comprehensive Resolver Tests — 10 test cases covering unqualified, qualified, scope, alias, and edge-case resolution

Technical Details

Added

  • Full Resolver implementation (src/import/resolver.*) — qualified and unqualified name lookup, scope management, alias following, side table recording
  • Stage::Resolved enum value in pipeline plan — new pipeline stage between Imported and TypeChecked
  • resolveStage() in CompilationSession — instantiates Resolver, runs it, extracts resolved table for sema
  • RangeNode AST node with lhs/rhs expression fields, builder method, and printer support
  • SymOrigin tracking in ImportManager — records which file and local symbol each merged symbol originates from (originOf() query)
  • SymbolTable::declareInScope() — declare a symbol in a specific scope (not just current)
  • Error codes NotNamespace (2005) and NoMember (2006) with human-readable descriptions
  • Windows _WIN32 guards in tool.cpp, run.cpp, project.cpp, interactive.cpp, deps.cpp
  • Resolver::takeResolvedTable() — moves out the resolved symbol side table for consumption by later stages
  • SemaPipeline::resolved_ pointer — optional reference to the resolver side table, checked before fallback lookup in visitIdent
  • 10 resolver unit tests in tests/unit/resolver-test.cpp

Changed

  • Resolved table integration — visitIdent checks the resolver side table first, falls back to syms().lookup(), and skips duplicate diagnostics when the resolver already reported an error
  • Import alias declaration now stores the target SymId in the symbol's target field, enabling alias chain resolution
  • Symbol table SymbolData struct — added SymId target field and updated constructor signature (8 parameters)
  • declare / declareInScope accept optional SymId target parameter
  • Parser — tryCompoundOp extended for =>, .. range detection before . field access, -> arrow member access
  • Diagnostics emit — fixed Optional(nullptr) to {}, added bounds checks on d.primary.end, clamped unsigned subtraction
  • visitIf in SemaPipeline — creates proper HirBranch node instead of discarding the then_branch value
  • CompilationSession constructor — initializes resolved_syms_ from sym_arena_
  • Import merge re-exports — restored duplicate-checking declare_re_export lambda

Fixed

  • Diagnostic emit — null optional crash in emit.cpp when optional holds nullptr
  • Diagnostic emit — bounds check on d.primary.end before access
  • Import merge — re-export path was silently overwriting duplicates instead of reporting DuplicateDecl
