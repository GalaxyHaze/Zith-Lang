v0.2.0 — The "We Wrote A Lot Of Code" Update

Released: June 2026 • Commits: 8 • Diff: +9849 / -4250 across 113 files

Well well well, look who's back for another update. This one's chonky. Almost ten thousand new lines of sweat, tears, and questionable life choices. Buckle up, bestie.

▸ +9849 lines written • ▸ -4250 lines deleted • ▸ 113 files violated • ▸ 4 new test suites born into this cruel world

✦ Parser Gets Glow-Up

  • parser +1700 lines of pure parse power. The scanner got a full rework with scan-result.hpp and parse-result.hpp — now with 47% more error recovery.
  • tests Brand new parser-full.cpp (857 lines) and parser-scan.cpp (269 lines) to make sure we didn't break everything.

✦ AST Gets a Personality

  • ast Dedicated AST nodes for enum, union, component, trait, and interface. No more shoving everything into generic declarations like a hoarder.
  • ast AST builder and printer both went through the washer — 670 lines of new structure, 500 of old dust removed.

✦ The "Did You Mean?" Engine

  • diag HeuristicEngine joins the squad! Levenshtein-powered suggestion engine that tells you "did you mean foobar?" when you typo'd fubar. Very judgmental.
  • diag Error codes got reorganized, emit system rewritten from the ground up. Your errors are now prettier and more helpful.

✦ Type System Goes to the Gym

  • types Type interning: completely rewritten (+202 lines). Faster, leaner, meaner.
  • types Unification engine expanded (+275 lines). New type kinds for components, traits, interfaces — because your types deserve more categories.
  • tests Two whole new test suites: type-dedup.cpp and unify-errors.cpp. We will find your bugs.

✦ Sema Gets Serious

  • sema Semantic analysis pipeline grew by 387 lines. Now properly type-checks enums, unions, traits, and interfaces. Your bad code has fewer places to hide.
  • tests New sema-hir.cpp (250 lines) to prove we mean business.

✦ Lexer Did a Speedrun

  • lexer Keyword table rebuilt from scratch (450 lines changed). Lexer logic restructured (650 lines). Tokens refactored. It's like we replaced the engine while driving.

✦ CLI Pipeline Unclogged

  • cli Compilation session massively refactored for the multi-stage pipeline. New stage enums, updated build commands, fresh CLI flags. Your zithc build experience is now chef's kiss.
  • cli Command handling cleaned up — build, run, deps, info all got some love.

✦ IR Trilogy (HIR → MIR → Tears)

  • zir HIR, MIR, and the lowering bridge between them all refactored. New types, cleaner insts, less pain (slightly).

✦ Memory Layer Spring Cleaning

  • memory Arena, DynArray, Optional, Result, Span, StringInterner, SourceFile, SourceMap — all got the consistency treatment. Same API patterns, fewer surprises. 750 lines of churn for your sins.

✦ Import/Export Logistics

  • import Import manager and resolver reworked to handle the new declaration types. Symbol table got a glow-up.
  • tests import-basics.cpp and import-e2e.cpp massively expanded (+490 lines total).

✦ Stdlib: The Unbreaking

  • stdlib Removed stdlib/unstable/json.zith. JSON support is on a journey of self-discovery and will return when ready.

★ Coming Up Next

  • More type checking — because there's never enough
  • Better error messages — your confusion is our priority
  • Actually compiling real programs (radical concept, we know)
