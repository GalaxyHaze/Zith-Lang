Contributing to Zith

Thank you for your interest in contributing to Zith! This guide will help you get started.

Ways to Contribute​

Report Bugs​

If you find a bug, please open an issue with:

  • A clear description of the problem
  • Steps to reproduce
  • Expected vs actual behavior
  • Your environment (OS, compiler version, etc.)

Suggest Features​

Open a feature request with:

  • The problem you're trying to solve
  • Your proposed solution
  • Any alternatives you considered

Write Code​

  1. Fork the repository
  2. Clone your fork locally
  3. Create a feature branch: git checkout -b feature/my-feature
  4. Make your changes
  5. Test your changes: cmake --build build
  6. Commit with clear messages
  7. Push to your fork
  8. Submit a Pull Request

Development Setup​

Prerequisites​

  • CMake 3.15+
  • C++17 compiler (GCC, Clang, or MSVC)
  • Git

Build from Source​

git clone https://github.com/GalaxyHaze/Zith.git
cd Zith
cmake -S . -B build
cmake --build build -j

Running Tests​

cmake -S . -B build -DBUILD_TESTING=ON
cmake --build build -j
ctest --test-dir build

Code Standards​

Formatting​

We use clang-format for code formatting. Run:

cmake --build build --target fmt

To check without modifying:

cmake --build build --target fmt-check

Commit Messages​

Use clear, descriptive commit messages:

  • Start with a verb (Add, Fix, Update, Remove)
  • Keep the first line under 72 characters
  • Add details in the body if needed

Pull Request Guidelines​

  • Keep PRs focused and reasonably sized
  • Include tests for new functionality
  • Update documentation if needed
  • Ensure all tests pass before submitting

Recognition​

Contributors will be acknowledged in the release notes and on our community page.

Questions?​

If you have questions, feel free to ask in GitHub Discussions or join our Discord.
