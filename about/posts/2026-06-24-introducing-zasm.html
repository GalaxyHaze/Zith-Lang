Introducing zasm — ZIR Assembler

2026-06-24

I started working on zasm — a standalone assembler for ZIR (Zith Intermediate Representation). It's still WIP, but already usable for small programs.

What it does

zasm takes .zas files and assembles them to ZIR bytecode, then runs them through a built-in interpreter. ZIR is the same IR that zithc targets, so zasm gives a direct path from assembly to execution without going through the full compiler pipeline.

Current state

Supported features:

  • Base arithmetic — add, sub, mul, div, rem (both int and float)
  • External C function calls via callext (FFI)
  • Branching — br, conditional branches (bz, bnz, bgt, ble, etc.)
  • Internal function calls with call — supports recursion (factorial, fibonacci)
  • Memory ops — load/store, push/pop, alloca, stm (store multiple registers)
  • Float support — fadd/fsub/fmul/fdiv, fld/fst, type conversion (ftoi, itof, etc.)
  • Constants via .const — integers, floats, strings, typed data
  • .extern for declaring external C functions
  • Macros with #define for textual substitution
  • Scoped labels in { } blocks — local to the enclosing function
  • Forward references — both for branches and function calls

Some examples. Arithmetic:

main:
    li      a0, 10
    li      a1, 32
    add     a0, a0, a1        ; a0 = 42
    add     ret, a0, zero
    ret

Recursive factorial:

main:
    li      a0, 5
    call    fact, 1
    ret

fact:
{
    push    a0
    li      t0, 1
    sub     t0, a0, t0
    ble     t0, base_case
    li      t0, 1
    sub     a0, a0, t0
    call    fact, 1
    pop     a0
    mul     ret, a0, ret
    ret
}

base_case:
    li      ret, 1
    pop     t0
    ret

Calling libc's write via FFI:

.const msg .asciz "hello from write\n"

main: (i32, *char, i64) > i64 {
    alloca a0, 48
    li t0, 3
    st t0, a0, 0            ; nargs = 3
    li t0, 1
    lea t1, db, msg
    li t2, 18
    stm a0, 8, t0, t1, t2   ; fd, buf, count
    callext write
    ret
}

How it fits

The assembler compiles to ZIR, which is also the target of zithc. This means small programs can mix both — write performance-critical hot paths in zasm and the rest in Zith, both reaching the same bytecode format.

Repository is part of the main Zith repo at lib/zasm/. Build with cmake -B build && cmake --build build. More examples in lib/zasm/examples/.

What's next

I want to make the syntax cleaner — looking at ways to reduce boilerplate for common patterns. The namespace rules for labels vs functions are solid but still need better error messages. ZIR emission is done; LLVM IR lowering from ZIR is the next big piece.
