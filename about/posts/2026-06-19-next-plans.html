Next Plans for Zith

2026-06-19

Here's what's on the roadmap for the next few weeks:

Short-term (v0.4.0)

  • Type system refinements — finish wiring the TypeWalker into all passes that currently duplicate the type-kind switch. The unification pass was just the first consumer.
  • Compilation session overhaul — the pipeline is getting more complex and the current CompilationSession needs better stage isolation and error recovery.
  • More comprehensive error reporting — the heuristic diagnostic engine is working but needs more "did you mean?" coverage for type mismatches and missing symbols.
  • Standard library expansion — basic collections (Array, List, Map) need to be usable from Zith code.

Medium-term

  • Self-hosted parser — the dream is to have the Zith parser written in Zith itself. This requires the FFI and bootstrapping infrastructure to be solid first.
  • MIR optimizations — the mid-level IR needs a proper optimization pass pipeline before we can generate competitive machine code.
  • Package manager maturity — zithc new, zithc build, and dependency resolution work but the UX is rough.

Long-term vision

  • LLVM backend stabilization — we have the IR, but codegen quality and platform coverage need significant work.
  • Language server (LSP) — the foundation exists but it's disabled. Proper IDE support will be a force multiplier for adoption.
  • Self-hosting — the ultimate milestone. A Zith compiler written in Zith.

As always, this is a solo project driven by curiosity and caffeine. Things move at the pace they move. But the direction is clear, and every release brings us a step closer.
