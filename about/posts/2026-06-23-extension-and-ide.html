Editor Tooling — VS Code Extension & Zith IDE

2026-06-23

Two editor-related projects got attention this week: the VS Code extension got some needed fixes, and I started a minimal standalone IDE for Zith.

VS Code Extension

The Zith extension on the marketplace has been updated. The main work was fixing the LSP binary download pipeline — download-lsp.js now pulls the correct platform-specific binary from the CI artifacts, and the extension correctly discovers it at startup.

Syntax highlighting, snippets, and the "Zith Dark" theme are all stable. The TextMate grammar covers all the latest keyword additions (component, union, scene, spawn, etc.) and the snippets match the current function, struct, and control flow syntax.

Zith-Studio

Beyond the extension, I've been experimenting with a small standalone IDE (placeholder name: Zith-Studio) — something lighter than VS Code but with deeper Zith integration. It's early days, but the idea is to have a zero-config environment where you just open a .zith file and get syntax highlighting, inline diagnostics from the LSP, and quick-run without any project setup.

Repository is up at GalaxyHaze/Zith-Studio. More details soon.
