zithc Improvements — Formatter, CAPI, Cross-Platform

2026-06-21

I've been hammering on zithc — the compiler itself — and a lot has landed since the last update. Three big areas got significant attention.

1. Code Formatter

The formatter is now a proper compiler pass, not an afterthought. A dedicated FmtVisitor (938 lines in fmt-visitor.cpp) walks every declaration, statement, and expression type with precedence-aware output and brace-block handling.

You can run it via zith fmt on any source file. It handles comma-separated lists, complex type expressions, semicolons vs braces, and nested blocks cleanly. There's also a companion test suite at tests/unit/fmt-visitor.cpp to keep formatting consistent across refactors.

2. C API (zithcLib)

The C ABI that the LSP consumes got major upgrades. The session lifecycle is solid — create from file or buffer, destroy, run the pipeline. New additions include:

  • Hover — zithc_hover() looks up an identifier at a byte offset and returns a markdown signature
  • Position conversion — zithc_offset_to_position() converts byte offsets to 0-based line/column pairs
  • Diagnostics with suggestions — the "did you mean?" heuristic engine and overload error enrichment are exposed for tooling
  • Cross-file symbol merging — merge_sym() preserves declaration IDs, spans, kinds, and targets across imports — groundwork for go-to-definition

The CAPI is now clean enough that I ripped out all the try/catch blocks in favor of nothrow new — the C boundary is pure C, no exception machinery needed.

3. Cross-Platform & Portability

The CI build matrix now covers Linux, Windows (MSVC + clang-cl), and macOS (x86_64 + ARM64). Getting there required a bunch of fixes:

  • Guarded unistd.h behind _WIN32 checks
  • Fixed isatty on Windows
  • Used xcrun on macOS for SDK path discovery
  • Switched Windows ARM64 to Zig cc for reliable cross-compilation
  • Fixed Token constructor for musl libc compatibility
  • Removed OBJECT library pattern that broke macOS universal binaries

The formatter and CLI now share a set of common file and terminal utilities, so new commands don't keep reinventing the same wheels.

v0.3.1 captured the critical bug fixes (TypeFn dangling span, empty-token-stream UB, lexer null fallthrough, SourceMap TOCTOU race), but the real headway is in the CAPI and formatter.
